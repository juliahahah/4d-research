**軸二:Distillation Method(怎麼蒸)**
- **Logit / response KD**:對齊最終輸出分布(soft label)→ 通常跟分類模型有關
- **Feature KD**:對齊中間層特徵圖 → 4D-RGPT 用的這種(LD)
- **Relation KD**:對齊樣本/模態間關係(RKD、CRD)→ 含我的跨模態關聯蒸餾:
  `L2( cos_sim(teacher_P_depth, teacher_P_flow), cos_sim(student_P_depth, student_P_flow) )`
  (物理根據:motion parallax,光流 ∝ 1/深度;提案前要補:①維度對齊(depth 1ch vs flow 2ch)②標量太稀→改逐 patch 相似度圖 ③先做 depth-flow 一對)

### 已知堵點
- **五種方法未定**:上面三類是候選,學長那套是否相同待確認——決定矩陣的行。
- **teacher 缺口**:L4P 只出 depth/flow/motion/camray;Scene flow、3D trace、RigidFlow、DINO 四列要自接 teacher(scene flow 估計器、CoTracker 類、GeoFlow 的 rigid-flow 計算、DINOv2)。
- **base 矛盾**:4D-RGPT code 的 base 是 NVILA-Lite-8B,驗證想用 Qwen2.5-VL——建議先在 NVILA 上復刻+加格,移植與否問老師。
- **配比**:原論文四模態同時蒸(λ 加權);逐格實驗=逐模態獨立蒸,變因要固定。
