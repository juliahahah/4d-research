> **用法**:此為草稿骨架。看完每頁「要講的」,闔上檔案自己講一遍,再回來對照有沒有漏。標 ★ 的是你的判斷,我不能替你填——那些空著上台一定被看穿。

上次 meeting 給了七個待辦,這次照順序回答。每題都講三件事:查到什麼、依據哪篇論文、還有什麼沒確認。中間會多三頁:一頁講最接近我們的先例 GeoFlow,兩頁是我設計的兩條路線,想請老師幫我選。

## P1|Q1:Flow4Agent 怎麼用 flow(60 秒)

**白話開場**:這篇教模型看一小時的長影片。影片太長塞不進模型,所以它用光流來「省」——會動的地方才重要,不動的丟掉。

**對著架構圖講**:
- 左到右:影片先切成一段段事件(TGO)——先用畫面變化粗切,再用光流確認「是真的有東西在動,不是燈光變了」。
- 進到留下的畫面裡,把不動的背景 token 剪掉,只留會動的前景(MTP)。
- 技術細節有人問再講:光流用 Sea-RAFT,切事件粗算 4 次、剪 token 精算 12 次;還會先扣掉鏡頭移動(homography),不然攝影機一晃全畫面都在「動」。

**右下消融表(這是重點)**:把光流模型整個換掉,總分只差 0.6。代表它對光流品質不敏感——光流在這裡只是配角。

**一句話收尾**:它把光流當**門票**(決定模型看什麼),從頭到尾沒參與訓練。我們想把光流變成**改考卷的筆**(決定模型學什麼)。

**誠實講**:論文沒做「整個拿掉光流」的對照,所以光流貢獻多少、去冗餘貢獻多少,拆不出來。

## P2|Q2:有沒有人把 depth/flow 放進 GRPO reward(60 秒)

**先給結論**:分兩邊講。**文字推理這邊:沒人做過**——這是我們的機會。**影片生成那邊:已經有很接近的了**,尤其 GeoFlow。

- 深度當 reward 很擠了:Smooth Operator、3D-R1、MetaSpatial(這篇編號查證過:2503.18470)……所以「深度」不新。
- 光流在推理側只被當輸入或剪枝工具;搜到一堆「Flow-GRPO」其實是 flow-matching——生成模型的數學,跟光流無關,別被名字騙。
- **主動講 GeoFlow**(下一頁展開):不主動講,審閱的人查到就是我們被抓包。

**必講的保留**:搜不到不等於不存在,我列了搜過的關鍵字,還要繼續掃引用鏈。

## P2b|GeoFlow 到底怎麼做(90 秒,對著它的原文架構圖)

**白話**:它訓練「影片生成」模型——生出來的影片如果動得不合理,就扣分。

**對圖走 A→B→C**:
- A:同一個題目生 G 支影片(GRPO 的群組比較)。
- **B(核心,講慢一點)**:用深度模型推算「假如場景都不動、只有鏡頭在動,畫面應該怎麼流」——這叫 rigid flow;再用光流模型量「實際怎麼流」。**兩個一減,差越大代表動得越不合理,分數越低。**會動的物體另外用特徵相似度處理。
- C:照 GRPO 更新,只在早期去噪步驟回傳梯度。

**收尾兩句(關鍵)**:
1. 差別——它的 reward 打在**影片像素**上,管「畫得對」;我們要打在**文字答案**上,管「想得對」。
2. 可以偷學——B 區那個「應該怎麼流 vs 實際怎麼流」的比對器,就是我們路線 B 需要的零件。

## P3|Q3:VLM 為何表現差(90 秒,最重要)

**白話問題**:模型答錯,到底是「看不清」還是「不會想」?

- 支持「看不清」:VLM4D 人類 98.8、最強模型 62.0,差 36 分,弱在跟蹤運動、保持時間一致。
- 支持「不會想」:MindCube 只微調語言那半(眼睛完全不動),分數就從 37.8 升到 51.4——幾乎全部的進步。但注意,它測的不是 4D 任務。
- **老實講:兩邊證據都有,而且沒有人在 4D 任務上做過拆解。這個缺口可能是我們自己做一個小實驗的機會。**
- MoCA 的辦法(右圖):讓模型把「看到的」寫成文字,丟給一個看不到圖的模型。解得出來=看的沒問題;解不出=看錯了。這個判定可以直接變 reward。

**必被問**:「所以你賭哪邊?」→ ★你的答案和理由:[自己填,講不出來就老實說還在收證據,但要說出打算怎麼收]

## P4|Q4:帶 depth 的資料集(60 秒)

**白話結論**:資料是夠的,真正要煩惱的是「授權」跟「偽標會錯」。

- 真的深度:ARKitScenes,雷射掃的,5 千段——**但授權非商用**(查過 Apple 的 LICENSE)。
- 合成的連光流真值都有:Spring(唯一可商用)、Virtual KITTI 2。
- 規模不用擔心:真值數萬幀、偽標(Depth Anything V2 自動生深度)要多少有多少。**規模不是瓶頸,授權才是**——研究用都沒問題。
- **偽標會錯,而且是文獻寫明的**:Depth Anything V2 是為單張圖設計的,跑影片會閃爍,給的還是「相對深度」不是公尺數。偽標錯,reward 就錯——這是我們方法必須正面設計對策的地方。

## P5|Q5:depth/flow 以外的模態(40 秒)

- 最貼 4D 的兩個:**scene flow**(光流的 3D 版,等於 depth+flow 合一)和 **3D 點追蹤**(跨畫面追同一個點,直接量時間一致性)。
- **老實講**:「它們比 2D 光流好」目前只是猜想,沒有實驗證據;偽標工具成熟度也還沒查。第一版我傾向 ★[2D flow / scene flow,自己選,理由是時程還是新穎性]。

## P6|Q6:reward 設計的文獻(50 秒)

- RLVR 一句話:用「程式能自動判對錯」的規則當 reward,便宜、不容易被騙。
- **通病**:答對 1 分答錯 0 分,太粗——「差一點就對」的答案什麼都學不到。
- 兩個解法:給過程分(中間步驟也算分)、給部分分(SNRA,下圖:差一點給 0.8 不是 0)。
- **主動講警訊**:有研究說 RL 只會放大模型本來就會的、教不出新能力——這對我們也是威脅,我們的回應是把 reward 打在感知上而不是只打在答案上(這正是兩條路線在做的事)。

## P7|Q7:賣點 + 我們的架構(60 秒)

- 唸候選賣點,馬上補一句:「這句還在改,以下是支持和反對。」
- 加分:深度那條路擠了、光流沒人走;考卷裡沒人考 flow 一致性;偽標讓 reward 可以無限算。
- 扣分(自己講,別等人問):根因沒定、偽標錯 reward 就錯、RL 可能教不出新能力、**最難的是 verifier——一句話跟一張光流場怎麼比,還沒有標準答案**。
- 右圖是我們的架構:上半是 GRPO 主迴路,下半兩條分支讓模型在 depth 圖和 flow 圖上答題、拿「完整性分數」。
- **一定要跑的對照**:光流直接當輸入做 SFT。如果那樣就追平,reward 的故事就不成立。

## P7b|路線 A:修補現有架構(90 秒)

**開場**:「接下來兩頁是我設計的兩條路,想請老師幫我看哪條更值得先走。」

**白話**:路線 A 不改故事、先把架構修穩。問題是:RL 只獎答案、不管感知——那就先教模型「讀懂」depth 和 flow。

**對圖講三個改動**(綠框=新增、橘框=修改,框上有來源論文):
1. 綠框:CSV 的題目改成**從標註自動生成的選擇/數值題**(學 Video-STR)——解決「題目跟標準答案哪來」。
2. 橘框:模型輸出改成「感知一段、推理一段」,獎懲**直接打到對應的 token**上(學 MoCA)——解決「輔助任務學的東西怎麼回到主任務」,不再靠一條模糊的線。
3. 橘框:判分規則化+差一點給部分分(學 Smooth Operator)——不需要請一個大模型當裁判,也就不會被騙分。

**被問「跟 4D-RGPT 的蒸餾差在哪」的標準答案**:蒸餾教模型「看得到」(改內部表徵),我們教模型「用得上」(改行為)。它自己論文承認蒸餾完推理還是不行——那正是 RL 的切入點。而且這兩件事可以拆開來量,這就是我的第一個實驗。

## P7c|路線 B:證據一致性 reward(90 秒)

**白話**:讀懂了不代表答題會用。路線 B 更直接——模型「想的」必須跟「世界實際怎麼動」一致,不一致就扣分。

**對圖講**:
1. 模型的感知段改成**固定欄位**:物體、方向、速度、方位(格式還在設計)。
2. 綠色大框「4D 證據 Verifier」,拆三塊:抽出模型的主張 → 偽標器算出深度和光流 → **用 GeoFlow 那套「應該怎麼流 vs 實際怎麼流」比對**,差多少給多少分。
3. 這個分數**只獎懲感知 token**(學 MoCA);原本的 CSV 降級成暖身課程。

**兩個漂亮的點**:
- 兩條 reward 互堵作弊:講模糊的話(「有東西在動」)→ 欄位抽不出來 → 完整性低分;亂講具體的話(「車往左」)→ 跟光流矛盾 → 一致性低分。
- 推論的時候完全不需要深度和光流(verifier 只在訓練用)→ 部署成本跟原模型一樣。

**誠實講風險**:verifier 是我們要發明的東西,偽標的錯會直接進 reward。所以驗證計畫第一條就是「verifier 跟人類判斷的一致率」(學 MoCA 報 86.31% 那樣)。

**收尾**:「A 穩、B 新;A 先走可以當 B 的暖身,兩條不互斥。我自己傾向 ★[你的立場+一句理由]——想聽老師的看法。」

## 附:名詞白話對照表(不看表講得出來才算過關)

| 術語 | 一句白話 |
|---|---|
| GRPO | 同一題答很多次,答得比同組平均好的加分、差的扣分——不用另外養一個裁判模型 |
| rollout | 同一題的其中一次作答 |
| advantage | 你比同組平均好多少(除以標準差) |
| RLVR | 用「程式能自動判對錯的規則」當 reward 的 RL |
| SNRA | 把「差一點就對」給部分分的轉換(誤差過 Sigmoid 變 0~1 連續分數) |
| credit assignment | 答錯時決定「該怪哪段輸出」——怪感知還是怪推理 |
| token 級信用分配 | 把獎懲精準打到「感知那幾個字」或「推理那幾個字」上,而不是整篇一起賞罰 |
| 蒙眼推理器 | 看不到圖的模型,只讀感知文字答題;它答得出=感知寫得夠好 |
| optical flow | 相鄰兩幀之間,每個像素往哪動、動多遠 |
| rigid flow | 假設場景全不動、只有鏡頭在動時,「理論上」的光流——跟實際光流相減就能抓出不合理的運動 |
| scene flow | 光流的 3D 版:每個點在空間中的移動向量 = depth+flow 合一 |
| 偽標 | 用現成大模型幫無標註資料自動生標籤(如 Depth Anything V2 生深度) |
| completeness reward(甲) | 模型在 depth/flow 上答題答得完不完整的分數 |
| consistency reward(乙) | 模型的說法跟光流證據合不合的分數 |
| 結構化主張 | 把感知寫成固定欄位(物體/方向/速度/方位),機器才驗得動 |
| MRA(相對準確率) | 數值題按誤差比例給分:答 2.1 公尺、正解 2.0,給高分不判死 |

## 附:最可能被問的五題(先想好)

1. **「A 還是 B,你自己選哪個?」**→ ★必須有立場才能去問老師。建議句式:「我傾向 X,因為 Y;但 Z 風險我還沒把握,想聽老師意見。」
2. **「跟 4D-RGPT 差在哪?」**→ 蒸餾教「看得到」、我們教「用得上」;它自承推理仍弱;可拆開驗證(MindCube 式)。
3. **「跟 GeoFlow 差在哪?」**→ 它 reward 打在像素(畫得對),我們打在文字(想得對);借它的比對器、不借它的任務。
4. **「為什麼不直接把 flow 當輸入餵給模型?」**→ 這正是必跑的 baseline,老實說:跑了才知道,這是第一個實驗。
5. **「訓練跟評測會不會用同批資料?」**→ 訓練:Spring/VKITTI2/偽標影片;評測:VLM4D/STI/R4D,不重疊。
