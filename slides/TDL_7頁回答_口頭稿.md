## P6|Q6:reward 設計的相關論文

**要講到的點**
- RLVR 一句話:用程式能自動判對錯的規則當 reward,便宜、不易被騙。
- 痛點:只看最終答案的 reward 太稀疏 → 兩個對策:過程 reward、連續化(指 SNRA 圖:差一點的答案拿部分分,不是零分)。
- **主動講警訊**:有研究說 RLVR 只放大既有行為、教不出新能力——這對我們也是威脅,不迴避。

## 附:名詞白話對照表(名詞白話測試用)

> 這是參考答案。**測試是你要不看這張表講得出來**——講不出的詞,回去讀,或從簡報刪掉。

| 術語 | 一句白話(參考) |
|---|---|
| GRPO | 一種 RL 訓練法:同一題讓模型答很多次,答得比同組平均好的加分、差的扣分,不需要另外訓練一個評分模型 |
| RLVR | 用「程式能自動判對錯的規則」當 reward 的 RL,例如答案對不對、格式對不對 |
| credit assignment | 答錯的時候,決定「該怪哪一段輸出」的問題——怪感知還是怪推理 |
| 蒙眼推理器 | 一個看不到圖的 LLM,只讀模型寫的視覺描述來答題;它答得出來,代表描述寫得夠好 |
| SNRA | 把「差一點就對」的答案給部分分數的轉換(用 Sigmoid 把誤差變成 0 到 1 的連續分數),讓 reward 不再全有全無 |
| MRA(相對準確率) | 數值題的計分法:答 2.1 公尺、正解 2.0,按誤差比例給分,不是判錯 |
| optical flow | 影片相鄰兩幀之間,每個像素往哪動、動多遠 |
| scene flow | 光流的 3D 版:每個點在三維空間的移動向量,等於 depth+flow 合在一起 |
| 偽標(pseudo-label) | 用現成大模型幫沒有標註的資料自動生標籤,例如用 Depth Anything 幫任意影片生深度圖 |
