# Pre-delivery checklist

Run through this before handing over a deck built with the presentation-style skill.

## Visual style
- [ ] Background is light (white / very light neutral), not dark, not pure black anywhere.
- [ ] Body text is dark blue-grey (`1F2A37`-ish), not pure black `000000`.
- [ ] Exactly one primary accent + one secondary color used throughout — not a rainbow.
- [ ] The deck doesn't read as flat black-on-white; the accent gives it life.

## Progress bar
- [ ] Present on every content slide, same position and size.
- [ ] Current chapter highlighted (solid accent, high contrast).
- [ ] Not-yet-reached chapters faded (light tint / low contrast).
- [ ] The current chapter is actually correct for each slide's content.

## Icons & diagrams
- [ ] Systems explained through diagrams, not bullet lists.
- [ ] Every brand/service icon is the real official one (fetched), in vendor standard colors.
- [ ] No hand-drawn or generated brand logos.
- [ ] No generic colored circles standing in for real icons.
- [ ] Any neutral stand-in boxes are listed for the user (which icons couldn't be fetched).

## Spotlight reveal
- [ ] Multi-part diagrams use a sequence of near-identical slides, one component lit per slide.
- [ ] Active component: full color/opacity + a highlight that's visible on white (accent, tint, or shadow — NOT a white outline).
- [ ] Other components: faded to pale grey.
- [ ] Element positions are pixel-identical across the sequence — nothing jumps.

## Progressive build
- [ ] Busy diagrams are assembled step by step, not dropped in fully formed.
- [ ] Color separates logical groups / legs of a flow.

## Analogy & text
- [ ] One running analogy carries the whole deck (not a new metaphor each slide).
- [ ] One idea per line; no long 透過…以…進而… clause chains.
- [ ] Concrete examples/numbers/pictures instead of abstract concept-naming.
- [ ] Everyday verbs, real subjects — not 優化/賦能-style officialese.
- [ ] Plain wording leads; jargon follows as a label.
- [ ] Few words per slide; detail lives in the spoken track.

## AI-tell avoidance
- [ ] No decorative line under any title.
- [ ] No edge color bars / stripes / sidebars.
- [ ] No leftover watermark/placeholder/filler cruft.
- [ ] Layout varies between slides (not "title + 3 bullets" every time).
- [ ] No abstract adjective walls without examples.
- [ ] No faded/light arrows on the light background — all lines dark enough to read.
- [ ] Titles centered; body text and lists left-aligned.
